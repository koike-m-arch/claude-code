# CWピン未読監視スキル

## 目的

Chatworkでピン止めしているルームの未読を定期チェックし、未読がある場合に知らせる。
ピン止めチャット＝優先度が高いため、見逃しを防ぐことが目的。

## ピン止めルーム一覧（固定）

以下のルームIDを対象とする（`list_rooms` の sticky フィールドは使用しない）：

| room_id | ルーム名 |
|---------|---------|
| 412599042 | 【グロース1課】GOAT |
| 397732275 | 【社内用】ブリリオ・ホワイト |
| 378851798 | 【チェルラーブリリオ】アルマード様×みらいきれい |
| 396651322 | 【チェルラーホワイト】アルマード様×みらいきれい |

## 実行手順

### Step 1: 各ルームのメッセージ取得

上記4ルームそれぞれに対して `mcp__chatwork__list_room_messages` を呼び出す。
- パラメータ: `force: 1`
- 4ルーム並列で実行する

### Step 2: 未読フィルタリング

取得したメッセージのうち、自分（小池勝也）が送信したメッセージを除外する。
自分のアカウントIDは `mcp__chatwork__get_me` で取得する。

未読メッセージが1件もないルームはスキップする。

### Step 3: 結果の表示

**未読ありの場合：**

```
## 🔔 ピン止めチャット 未読あり！（HH:MM時点）

**[ルーム名]**
👤 [送信者名]
> [メッセージ冒頭100文字]...
📅 [受信日時]
⚡ 早めに返信しましょう！

---
📌 ピン未読: X件
```

**未読なしの場合：**

何も表示しない。通知不要。ユーザーの作業を妨げないようにサイレントで終了する。

## 定期実行

このスキルは `/loop` と組み合わせて使う。

**10分ごとに監視する場合：**
```
/loop 10m /CWピン未読監視
```

Claude Codeのセッションが続いている間、10分ごとに自動チェックされる。

## 注意事項

- ピン止めルームが0件の場合は「ピン止めされたルームが見つかりません」と表示する
- メッセージ取得に失敗したルームはスキップしてよい
- 自分が最後に送信したルームは「返信済み」として扱い、未読であっても通知不要
