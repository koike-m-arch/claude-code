# 日報作成スキル

## 目的

毎日の日報を対話形式で作成し、Chatworkの「【日報】グロース1課_マーケティング事業部」（room_id: 414660007）に投稿する。

## 投稿フォーマット

```
[toall]
▼日報の項目
・今日のひとこと
　{今日のひとこと}

・成果目標
・{M}月 Bヨミ
グロース1課：¥XX,XXX,XXX/¥XX,XXX,XXX　XX.X%
個人　　　  ：¥XX,XXX,XXX/¥XX,XXX,XXX　XX.X%

・今日のゴール
∟{タスク1}
∟{タスク2}

・今日の成果
∟上記を完遂した。
　　
・学び
∟{学びの要約}

・明日のゴール
∟{タスク1}
∟{タスク2}
```

## 実行フロー

### Step 1: Googleカレンダーから今日・明日の予定を取得

`mcp__google-workspace__get_events` を**今日と明日の2回**呼び出す（並行実行）。

- **user_google_email**: `koike-m@mirai-kirei.jp`
- **今日**: `time_min` = 今日の00:00:00+09:00、`time_max` = 今日の23:59:59+09:00
- **明日**: `time_min` = 明日の00:00:00+09:00、`time_max` = 明日の23:59:59+09:00
- **max_results**: 20

取得後、以下のルールで「今日のゴール」「明日のゴール」を生成する：

**除外するもの（定例・ルーティン）**
- 日報提出
- 管理画面チェック
- 運用調整・制作
- 朝会
- 週次MTG（荒屋敷さんとの定例など、週次の定例MTG全般）
- 週次アンケート記入
- その他、明らかに毎週・毎日繰り返される定例イベント

**含めるもの（例外・スポット）**
- 週次レポート作成（例：ブリリオレポート）→ **必ず含める**
- クライアント・外部との打ち合わせ（例：尾藤さま打ち合わせ）
- 撮影、黒船壁打ち、あんこプロジェクトなどスポット業務
- 社内イベント・祝勝会など特別なもの

- イベント名から★や▼などの記号は除いてよい

### Step 2: ユーザーへの質問（Step 1と並行実行）

成果目標の取得（Step 3）と同時に、**学びのみ**を聞く：

```
日報を作成します！カレンダーから今日・明日の予定を取得中です。

②学び・振り返りを教えてください
（自由に書いてください。箇条書きでも長文でもOK。複数あればまとめて書いてください）
```

### Step 3: ユーザーの回答を受け取ったら日報を組み立てる

ユーザーから①②③の回答が届いたら、以下の順で組み立てる。

#### 今日のひとこと（自動生成）
- ユーザーの今日のゴール・学び・明日のゴールの内容を踏まえて生成する
- 実際の過去例の雰囲気（カジュアル、個人的な一言、ユーモアあり）を参考にする：
  - 「ブリリオの限界を突破しにかかりたい」
  - 「今日はビックTAV」
  - 「今日は雨が降るらしいからジムに行く」
- 20〜40文字程度の短い一言にする

#### 今日のゴール（整形）
- ユーザーが入力したタスクを各行の頭に `∟` をつけて整形する
- 例：`∟ブリリオ数値集計` `∟ホワイトテコ入れ`

#### 今日の成果（固定）
- 常に `∟上記を完遂した。` を入れる
- その後に全角スペース2文字の空行（`　　`）を入れる

#### 学び（要約・整形）

ユーザー（小池勝也）の実際の文体・口調を正確に再現すること。以下を厳守する。

**構成の流れ**
体験の描写 → 感情・反応 → 気づき・考察 → 決意・行動宣言

**フォーマット**
- 書き出しは `∟本日は〜` で始める
- **句点（。）のたびに改行する**。次の行頭は全角スペース（　）でインデントする
- 300〜500文字程度

**口調・表現の特徴（必ず再現する）**

① 柔らかい断定・推測表現を多用する
　→「〜んだと思う」「〜気がする」「〜なんだろう」「〜のかもしれない」「たぶん」「多分」

② 感情が強い場面では口語・俗語も使う
　→「たまらん」「〜ぞ」「マジで」「バチバチに」「ざま」「〜てくれる」「いっぱいいっぱい」

③ 問いを立てて自己対話する形式
　→「〜とは何か？考えた。猛省に猛省を重ね。出た。」のように問答を展開する

④ 固有のエピソード・引用をそのまま活かす
　→ 野球体験談、アニメ（ハイキュー・サマーウォーズ）、名言（孫氏など）が出てきたら原文の言葉を残す

⑤ 失敗・反省を前向きに締める
　→「失敗してよかった」「糧にする」「来月は成功する」「超えていく」のように必ずポジティブな決意で締める

⑥ 自己成長の実感を素直に表現する
　→「少しづつでも成長をしているぞ」「やっとできるようになってきた」など

⑦ アクション・気づきを箇条書き（①②③）でまとめることもある
　→ 定例振り返りや次回改善点を整理するときに使う

**NGパターン（やってはいけない）**
- ビジネス文書調・丁寧すぎる文体（「〜と感じました」「〜と思われます」）
- 評論家的な表現（「一般的に〜」「本質的には〜」）
- 結論を先に言う構成（体験から入ること）
- 原文にないエピソードや引用を追加する

#### 明日のゴール（整形）
- ユーザーが入力したタスクを各行の頭に `∟` をつけて整形する

### Step 4: 下書きを表示

完成した日報の下書きをコードブロックで表示し、以下を添える：

```
上記が日報の下書きです。内容を確認して「送信」と言ってください。
修正がある場合はお知らせください。
```

### Step 5: 送信

ユーザーが「送信」「OK」「これで送って」「いいです」などの承認の言葉を言ったら、
`mcp__chatwork__post_room_message` を使って投稿する。

- **room_id**: `414660007`
- **body**: Step 4で組み立てた日報の全文（コードブロックなし、プレーンテキストで）

投稿後に表示するメッセージ：
```
✅ 日報を「【日報】グロース1課_マーケティング事業部」に送信しました！お疲れ様でした。
```

## 注意事項

- 「∟」記号は「ルート記号（∟）」であり、バックスラッシュや「└」とは異なる。必ず `∟` を使うこと。
- 成果目標の数値が0円やエラーの場合は「取得できませんでした（確認してください）」と記載してユーザーに知らせる。
- ユーザーが修正を求めたら、修正後に再度下書きを表示してから送信確認をとる。
- 毎回必ず粗利スプレッドシートからデータを取得する（キャッシュ使用不可）。
