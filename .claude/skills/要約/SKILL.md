# 要約スキル

## 目的

貼り付けられたテキストを素早く整理し、「何が書いてあるか・何が重要か・次に何をすべきか」を一目で把握できる形にまとめる。読む手間を最小化し、即アクションにつなげることが目標。

## 出力フォーマット

必ず以下の構成で出力すること。見出しの絵文字・スタイルはこのまま使う。

```
## 📋 3行要約
1. （全体の内容を端的に1文で）
2. （主な課題・状況を1文で）
3. （重要な結論・方向性を1文で）

---

## 🔑 ポイント5つ
1. **[ポイントのタイトル]** — 説明
2. **[ポイントのタイトル]** — 説明
3. **[ポイントのタイトル]** — 説明
4. **[ポイントのタイトル]** — 説明
5. **[ポイントのタイトル]** — 説明

---

## ✅ ネクストアクション
- [ ] （誰が・何を・いつまでに、の形で具体的に）
- [ ] （同上）
- [ ] （同上）
```

## 各セクションの書き方

### 📋 3行要約
- 1行目：文章全体のテーマ・目的
- 2行目：現状の課題・状況・背景
- 3行目：結論・方向性・重要な決定事項
- 1行30〜50文字程度を目安に簡潔に

### 🔑 ポイント5つ
- 文章の中で特に重要な情報・数値・決定事項を抽出する
- 太字のタイトル（3〜8文字）＋ダッシュ＋1文の説明
- 数値が含まれる場合は必ず数値を記載する
- 重要度の高い順に並べる

### ✅ ネクストアクション
- 文章から読み取れる「やるべきこと」を具体的に書く
- 「誰が・何を・いつまでに」が明確になるよう記述する
- 明確な担当・期限が読み取れない場合は「要確認」と記載する
- アクションが読み取れない文章の場合は「特になし（情報共有のみ）」と記載する

## 文体・トーン
- 箇条書きは体言止めを基本とする
- 専門用語はそのまま使用してよい（KPI・CVR・CPA・ROAS 等）
- 社内向け文章はカジュアル、クライアント資料ベースの文章は丁寧に

## 注意事項
- テキストが貼られていない場合は「要約したい文章を貼り付けてください」と一言で促す
- 文章が短すぎる（3行未満など）場合も同じフォーマットで出力する（ポイントは3つ以上で可）
- 複数の文章が連続して貼られた場合は、それぞれ個別に要約するか確認する

## 使用例

**例1：**
ユーザー: `/要約 [長文テキスト貼り付け]`
→ フォーマット通りに即座に要約を出力する

**例2：**
ユーザー: `この議事録まとめて [テキスト]`
→ 同じフォーマットで出力する

**例3：**
ユーザー: `/要約`（テキストなし）
→「要約したい文章を貼り付けてください！」と促す
