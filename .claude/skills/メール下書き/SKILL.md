# メール下書きスキル

## 目的

相手と用件を伝えるだけで、すぐに使えるビジネスメールの下書きを作成する。ユーザーの手間を最小化しつつ、相手に好印象を与える自然な敬語・丁寧な文体で仕上げることが目標。

## 情報の収集

引数や会話の文脈からできる限り情報を読み取ること。以下の情報を把握した上でメールを作成する。

| 項目 | 内容 |
|------|------|
| 宛先 | 相手の名前・会社名・部署など |
| 用件 | メールの目的（依頼・連絡・お礼・確認・謝罪・提案 など） |
| 背景・詳細 | 用件の補足情報（あれば） |
| トーン | 特に指定がなければ「丁寧・自然」で作成 |

情報が不足している場合は、**一度に必要な情報をまとめて質問する**（何度も聞き返さない）。ただし、ある程度の情報があれば積極的に下書きを作成して見せ、後から修正する方が効率的なことも多い。

## メールの構成

必ず以下の構成で作成すること。

```
件名：[用件が一目でわかる簡潔なタイトル]

[会社名・宛名] [敬称]

いつもお世話になっております。
みらいきれいの小池でございます。

[用件の本文]
※簡潔かつ丁寧に。要点を明確に伝える。
※長くなる場合は箇条書きも活用する。

[締めの一文]
※「ご確認のほど、よろしくお願いいたします。」など用件に合わせて

何卒よろしくお願いいたします。

━━━━━━━━━━━━━━━━━
株式会社みらいきれい
マーケティング部
小池 勝也（こいけ まさや）
━━━━━━━━━━━━━━━━━
```

## トーン・文体のガイドライン

- **丁寧だが堅すぎない**自然な敬語を使う
- 「〜でございます」「〜いたします」を適切に使い、くどくならないよう注意
- 一文を短くまとめ、読みやすさを優先する
- 用件に応じてトーンを調整する：
  - お礼・挨拶 → 温かみのある文体
  - 依頼・確認 → 簡潔・丁寧
  - 謝罪 → 誠実さを前面に
  - 提案・報告 → 明確・論理的

## 作成後の確認

下書きを提示したら、必ず最後に以下を添える：

> この内容でよろしいでしょうか？修正があればお知らせください。

## 使用例

**例1（引数あり）：**
ユーザー: `/メール下書き 柴田さん宛に、新しいLPの制作依頼メールを書いて`
→ 柴田さん宛、LP制作依頼の下書きをすぐに作成する

**例2（会話から情報を読み取る）：**
ユーザー: `さっきの件でアルマードの北口さんにお礼メールを送りたい`
→ 会話の文脈から「さっきの件」の内容を把握し、お礼メールを作成する

**例3（情報不足の場合）：**
ユーザー: `/メール下書き`
→「宛先と用件を教えていただければ、すぐに下書きを作成します！」と一言で確認する
